## Embeddings Component Gets Flexible: Metadata & Keboola Table Output
Our embedding generation component now supports adding metadata columns from your source data and writing results directly to Keboola Tables alongside vector database destinations.

The component responsible for generating text embeddings using sentence-transformers has received significant updates, offering greater flexibility and richer data output.

You can now select multiple columns from your input table to be included as metadata alongside the generated embedding vector and the source row ID. This allows you to store crucial context directly with your embeddings, whether you're sending them to a vector database like Pinecone or storing them elsewhere.

Furthermore, we've introduced the ability to write the output – including the ID, original text, embedding vector, and any selected metadata – directly to a Keboola Storage Table (CSV format). This provides a straightforward way to utilize embeddings within the Keboola ecosystem without requiring an external vector database.

For users sending data to Pinecone, the component now uses an `upsert` operation. This means existing vectors with the same ID will be updated, preventing duplicates and allowing for easy refreshes of your embedding data. We've also increased the processing limits to better handle very large text fields in your input data.
