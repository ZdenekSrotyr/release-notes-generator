## Supercharge Your Embeddings: Qdrant, Native Metadata & Pinecone Updates
Our embeddings generation component now supports Qdrant, preserves metadata types in vector databases, and features enhanced Pinecone integration.

We've significantly updated our component for generating sentence embeddings from your Keboola tables. These enhancements provide more flexibility in choosing your vector storage and enrich the data you store.

Key updates include:
*   **Qdrant Destination Support:** You can now send your generated embeddings and associated metadata directly to a Qdrant vector database collection, offering another powerful option alongside Pinecone and CSV output.
*   **Native Metadata Types:** When sending data to vector databases (Pinecone and Qdrant), metadata columns selected from your source table now retain their original data types (like integer, float, boolean) instead of being converted to strings. This allows for more precise, type-specific filtering and querying within your vector database.
*   **Enhanced Pinecone Integration:**
    *   **Auto-Creation:** The component can now automatically create the target Pinecone index if it doesn't exist, simplifying setup.
    *   **Upsert Operation:** We now use Pinecone's `upsert` functionality, meaning records with existing IDs will be updated rather than causing errors or duplicates, ensuring idempotent write operations.
*   **Improved Robustness:** We've increased the processing limit for large text fields in input CSVs and refined the output structure definition for CSV destinations.

These updates make generating and storing embeddings more powerful and user-friendly, enabling richer context storage and more flexible vector search applications.
